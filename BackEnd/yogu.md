# 비대면 대출 신청 서비스 요구사항 명세서

## 1. 회원가입

### 1.1 회원가입

- 사용자는 기본정보를 입력하여 회원가입을 진행할 수 있어야 한다.
- 입력 항목
    - 이름
    - 휴대전화번호
    - 생년월일
    - 주소
    - 이메일

### 1.2 이메일 인증

- 사용자는 이메일 인증을 완료해야 회원가입을 진행할 수 있어야 한다.

### 1.3 간편인증 비밀번호 등록

- 사용자는 로그인에 사용할 간편인증 비밀번호를 등록할 수 있어야 한다.

### 1.4 계좌 생성

- 회원가입 완료 시 입출금 계좌가 자동 생성되어야 한다.
- 사용자는 계좌 비밀번호를 설정할 수 있어야 한다.

### 1.5 전자서명 비밀번호 등록

- 사용자는 전자서명에 사용할 비밀번호를 등록할 수 있어야 한다.

---

## 2. 로그인

### 2.1 로그인

- 사용자는 등록한 간편인증 비밀번호를 이용하여 로그인할 수 있어야 한다.
- (id, pw 로그인)

---

## 3. 대출 신청 사전 절차

### 3.1 대출상품 선택

- 사용자는 신청 가능한 대출상품을 조회하고 선택할 수 있어야 한다.

### 3.2 대출 신청서 생성

- 사용자가 대출 신청을 시작하면 신청서가 생성되어야 한다.
- 신청서는 채번되어 관리되어야 한다.
- 신청서는 생성일 기준 1일 동안 유지되어야 한다.

### 3.3 적합성·적정성 확인

- 사용자는 적합성·적정성 고객정보 확인서를 작성할 수 있어야 한다.
    - 대출목적, 상환방법, 자산규모, 연간소득 입력
- 시스템은 입력된 정보를 저장할 수 있어야 한다.

### 3.4 본인인증

- 사용자는 본인인증을 수행할 수 있어야 한다.
    - 간편비밀번호 사용
- 인증 완료 시 적합성·적정성 확인 절차를 완료 처리해야 한다.
    - (적부 여부는 어떻게 결정)

### 3.5 공공마이데이터 이용 동의

- 사용자는 본인 행정정보 제공 요구서에 동의할 수 있어야 한다.
- 사용자는 공공마이데이터 활용에 동의할 수 있어야 한다.

### 3.6 약관 및 동의서 열람

- 사용자는 아래 문서를 조회할 수 있어야 한다.
    - 본인 행정정보 제공 요구서
    - 개인(신용)정보 수집·이용·제공 동의서

### 3.7 전자서명 인증

- 사용자는 전자서명을 통해 본인확인을 완료할 수 있어야 한다.

---

## 4. 대출 한도 조회

### 4.1 휴대폰 본인인증 약관 동의

- 사용자는 휴대폰 본인인증 관련 약관에 동의할 수 있어야 한다.

### 4.2 전자서명 토큰 발급

- 사용자는 인증수단을 선택할 수 있어야 한다.
- 시스템은 전자서명용 토큰을 발급할 수 있어야 한다.
- 공동인증서 비밀번호 입력을 통해 인증을 수행할 수 있어야 한다.

### 4.3 직장 및 소득정보 입력

- 사용자는 직장정보 및 소득정보를 입력할 수 있어야 한다.

### 4.4 공공마이데이터 조회

- 시스템은 공공마이데이터를 조회할 수 있어야 한다.
- 사용자는 조회 결과를 확인할 수 있어야 한다.

### 4.5 대출한도 산출

- 시스템은 공공마이데이터 기반으로 대출심사를 수행할 수 있어야 한다.
- 시스템은 최대 대출한도와 적용 금리를 산출할 수 있어야 한다.
    - (산출 로직)
- 사용자는 산출 결과를 확인할 수 있어야 한다.

### 4.6 상품설명 제공

- 시스템은 금융상품 중요사항 설명을 제공할 수 있어야 한다.ㅁ
- 사용자는 설명 내용을 확인할 수 있어야 한다.

---

## 5. 대출 약정 및 실행

### 5.1 약관 동의

- 사용자는 약관, 설명서 및 관련 문서를 확인할 수 있어야 한다.
- 사용자는 각 문서에 동의할 수 있어야 한다.

### 5.2 대출 조건 입력

- 사용자는 다음 정보를 입력할 수 있어야 한다.
    - 대출상환방식
    - 우대금리 항목
    - 신청금액
    - 기준금리
    - 대출기간
    - 리프라이싱 주기

### 5.3 입금계좌 및 자금용도 입력

- 사용자는 대출금 입금계좌를 입력할 수 있어야 한다.
- 사용자는 자금용도를 입력할 수 있어야 한다.

### 5.4 신청정보 확인

- 시스템은 대출 신청정보를 최종 확인 화면으로 제공할 수 있어야 한다.
- 사용자는 신청정보를 검토할 수 있어야 한다.

### 5.5 본인확인

- 사용자는 전자서명을 통해 본인확인을 수행할 수 있어야 한다.

### 5.6 전자약정

- 사용자는 채권 관련 전자약정을 체결할 수 있어야 한다.

### 5.7 최종 본인인증

- 사용자는 최종 본인인증 절차를 수행할 수 있어야 한다.

### 5.8 대출 실행

- 시스템은 대출 실행 조건 충족 시 대출을 실행할 수 있어야 한다.
- 시스템은 승인된 대출금을 지정 계좌로 입금할 수 있어야 한다.
1. 회원가입 ( 계좌 자동 생성 )
기본정보만 받고(이름 전화번호 생년월일 주소 이메일인증→간편인증비밀번호 계좌-계좌비밀번호 전자서명에 사용할 비밀번호)
2. 로그인
간편인증비밀번호로 로그인
3. 대출신청
대출 선택 → 대출 신청(이때 신청서 채번 등록 하루 유지) → 적합성 적정성 고객정보 확인서 입력 → 본인인증 → 적합성 적정성 확인 완료 → 본인 행정정보 제공 요구서 및 공공마이데이터 동의 → 본인 행정정보 제공 요구서 뷰잉 → 본인 행정정보 제공 요구서 뷰잉 → 개인(신용)정보 수집이용제공 동의서 뷰잉 → 개인(신용)정보 수집이용제공 동의서 뷰잉 → 휴대폰 본인인증(대신에 전자서명)
4. 대출 신청
휴대폰 본인인증 약관 동의 → 전자서명(토큰발급)_인증선택 → 전자서명(토큰발급 → 공동인증서 비밀번호) → 직장소득정보 입력 → 공공마이데이터 확인 화면 → 1만원이라도 높은 한도의 화면 → 대출한도조회 화면(공공마이데이터 수집정보 기반 대출심사_최대한도 및 금리 산출) → 금융상품에 대한 중요사항 설명1 → 금융상품에 대한 중요사항 설명2
5. 대출 신청 
금융상품에 대한 중요사항 설명1 → 금융상품에 대한 중요사항 설명2 → 약관, 설명서 등 확인 및 동의1 → 약관, 설명서 등 확인 및 동의(이미지 여러장) → 대출상환방식 입력 → 우대금리 항목 입력 → 신청금액, 기준금리, 대출기간, 리프라이싱 주기 입력→ 대출금 입금계좌, 자금용도 입력 → 대출 입력(심사)정보 확인 및 약정 진행 → 본인확인(대신에 전자서명) → 채권서류 전자약정 → 본인인증 → 대출 실행
